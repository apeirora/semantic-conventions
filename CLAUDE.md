# CLAUDE.md — apeirora/semantic-conventions

OpenTelemetry Semantic Conventions repository (fork of open-telemetry/semantic-conventions).
Defines attributes, metrics, and events for logs, traces, and metrics via YAML + Markdown.

## Repo structure

```
model/          YAML source of truth — attribute definitions, events, metrics, spans
  <area>/
    registry.yaml   attribute definitions (id, type, stability, brief, note, examples)
    logs.yaml       event/log groups referencing registry attrs with requirement levels
    metrics.yaml    metric definitions
    spans.yaml      span definitions
docs/           Markdown documentation (tables auto-generated by Weaver from model/)
  <area>/
  registry/attributes/<area>.md   per-namespace attribute reference (FULLY auto-generated by weaver — never edit by hand)
.chloggen/      One YAML file per PR for changelog tracking
.github/
  workflows/    GHA jobs (see below)
  ISSUE_TEMPLATE/  auto-generated from model/ area names
templates/      Weaver Jinja2 templates for markdown generation
policies/       OPA Rego policies checked by Weaver
```

## Mandatory workflow after any change

**Every AI-assisted change MUST run these steps before committing, in order:**

```bash
# 1. Regenerate all tables and registry docs from the YAML model
make table-generation        # updates semconv snippet tables inside docs/**/*.md
make registry-generation     # regenerates docs/registry/attributes/<area>.md

# 2. Sync GH issue template area dropdowns
make generate-gh-issue-templates   # adds new model/ area names to .github/ISSUE_TEMPLATE/

# 3. Auto-fix markdown
make misspell-correction     # fix spelling
make markdown-toc            # update <!-- START doctoc --> tables of contents

# 4. Verify everything passes (runs all GHA checks locally)
make table-check             # dry-run Weaver — must produce no diff
make check                   # markdownlint + misspell + toc-check + link-check + policies + registry-generation
```

Or run everything in one shot:

```bash
make fix && make generate-gh-issue-templates && make check
```

## GHA jobs and their local equivalents

| GHA job | Local command | Fails when… |
|---|---|---|
| `markdownlint` | `make markdownlint` | Markdown style violations |
| `yamllint` | `make yamllint` | YAML style violations |
| `misspell` | `make misspell` | Spelling errors in `.md` files |
| `markdown-toc-check` | `make markdown-toc-check` | `<!-- START doctoc -->` sections out of date |
| `link-check` | `make markdown-link-check-local-only` | Broken links |
| `semantic-conventions` | `make table-check` | Semconv tables in `docs/` differ from what Weaver would generate from `model/` |
| `semantic-conventions-registry` | `make registry-generation && git diff --exit-code './docs/registry/*.md'` | `docs/registry/attributes/*.md` out of date |
| `areas-dropdown-check` | `make generate-gh-issue-templates && git diff --exit-code '.github/ISSUE_TEMPLATE'` | Area dropdowns in issue templates out of date |
| `schemas-check` | `make schema-check` | Schema files invalid |
| `policies-check` | `make check-policies` | Weaver OPA policy violations |
| `polices-test` | `make test-policies` | OPA policy unit tests fail |
| `dead-yaml-check` | `make check-dead-yaml` | YAML signal groups defined but not referenced in any `.md` |

| `changelog` | `make chlog-validate` | No `.chloggen/*.yaml` added, or entry invalid |

## Adding a new semantic convention area

When a new `model/<area>/` directory is created, these files are affected automatically by the make targets above — but they need to be committed explicitly:

1. `docs/<area>/*.md` — written/updated by `make table-generation`
2. `docs/registry/attributes/<area>.md` — **fully overwritten** by `make registry-generation`; never create or edit manually
3. `.github/ISSUE_TEMPLATE/*.yaml` — updated by `make generate-gh-issue-templates`
4. `.chloggen/<branch-or-area>.yaml` — must be created manually (see below)

## Changelog entry (required for every non-chore PR)

Every PR that touches user-facing conventions needs a `.chloggen/*.yaml` entry. PRs with `[chore]` in the title or the `Skip Changelog` label are exempt.

```bash
make chlog-new    # creates .chloggen/<current-branch>.yaml from TEMPLATE
```

Fields:
- `change_type`: one of `breaking`, `deprecation`, `new_component`, `enhancement`, `bug_fix`
- `component`: the `model/` area name (e.g. `audit`, `http`, `db`)
- `note`: one-line description
- `issues`: list of integers (PR number is fine if no issue exists)
- `subtext`: optional multi-line detail (pipe `|` syntax)

## YAML model conventions

- Attributes live in `model/<area>/registry.yaml` under `groups[].id: registry.<area>.*`
- Requirement-level groups live in `model/<area>/logs.yaml`, `spans.yaml`, etc. and use `ref:` to reference registry attributes
- `stability` values: `stable`, `release_candidate`, `development` (never omit)
- Enum types use a `members:` list, each member needs `id`, `value`, `brief`, `stability`
- `note:` is rendered verbatim into the generated table footnotes — keep it precise
- Resource-level attributes (e.g. `audit.integrity.algorithm`) must be documented as such in their `note:` field

## Markdown conventions

- Semconv tables inside `docs/**/*.md` are bounded by `<!-- semconv <group-id> -->` / `<!-- endsemconv -->` — **do not edit these blocks by hand**, they are overwritten by `make table-generation`
- Hugo front matter goes at the very top as an HTML comment: `<!--- Hugo front matter ... --->`
- File and folder names under `docs/` must use hyphens, never underscores (`check-file-and-folder-names-in-docs` job)
- Every `docs/` page with more than one section needs a `<!-- START doctoc -->` / `<!-- END doctoc -->` block

## Tools required locally

- Docker or Podman (for Weaver and OPA containers — versions pinned in `dependencies.Dockerfile`)
- Node.js + npm (for markdownlint, doctoc)
- Go (for misspell, chloggen)

Install Node.js tools: `npm ci --ignore-scripts`
